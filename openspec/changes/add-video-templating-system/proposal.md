# Change: 视频模板化自动渲染系统

## 为什么
- 需要一套可复用的模板化视频生成能力，通过 Remotion 自动化渲染，减少手工操作。
- 使用 Strapi 统一管理模板、素材与实例，支持后续扩展队列、状态回填。

## 将要修改的内容
- 定义视频模板/实例/素材的后台模型与校验（Strapi）。
- 设计渲染输入输出契约，基于 Remotion 执行渲染并回填产物链接与状态。
- 搭建基础渲染任务流（等待/渲染中/完成/失败），支持后续接入队列和重试。

## 影响
- 受影响模块：Strapi 模型与 API、Remotion 渲染入口/脚本。
- 可能的破坏性变更：无（新增能力）。
